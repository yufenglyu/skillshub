## SkillsHub 0.11.1

This maintenance release makes skill ownership predictable across the Resource Library, Central Skills, and individual platforms.

### Highlights

- Installing a Resource Library skill directly to a selected platform now affects only that platform. It no longer creates a Central Skills copy or propagates to unrelated tools.
- Promoting a skill to Central Skills remains the explicit way to share it, and now synchronizes it to enabled and detected platforms.
- Platform views only list installations managed by SkillsHub; platform-native skills that are not in the Resource Library are left untouched and hidden from managed counts.
- Platform cards now identify Resource Library, Central Skills, and standalone sources correctly, including symlink or copy mode.
- Local ZIP and WebDAV backups always include Resource Library files, Central Skills files, app configuration, and platform installation state. Skill files are stored directly in the archive instead of being embedded in one JSON file.
- WebDAV connection settings persist locally, while passwords, tokens, and API keys remain excluded from backup archives.
- Central Skills counts, Codex CLI directory handling, and managed installation state are preserved across scans and restarts.
- The desktop icon, documentation, and application screenshots have been refreshed. Unused mobile icon assets were removed.

## SkillsHub 0.11.1 中文说明

本维护版本明确区分技能资源库、中央技能库和单个平台安装，避免技能被意外同步到其他软件。

### 主要更新

- 从技能资源库直接安装到指定平台时，只影响所选平台，不会再进入中央技能库，也不会扩散到其他软件。
- “加入中央技能库”仍是共享技能的明确操作；加入后会自动同步到已启用且已检测到的平台。
- 平台页只展示由 SkillsHub 管理的安装；软件自身安装、且不在技能资源库中的 skills 不纳入列表和统计，也不会被修改。
- 平台技能卡片会正确区分技能资源库、中央技能库和独立安装来源，并显示符号链接或复制方式。
- 本地 ZIP 与 WebDAV 备份固定包含技能资源库文件、中央技能库文件、软件配置和平台安装状态；技能目录以文件形式直接写入归档，不再全部塞入单个 JSON。
- WebDAV 连接配置会保存在本机，密码、Token 和 API Key 仍不会写入备份归档。
- 修复中央技能库统计、Codex CLI 目录识别，以及扫描和重启后的安装归属保持问题。
- 更新桌面图标、说明文档和应用截图，并移除不需要的移动端图标资源。
