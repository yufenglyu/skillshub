# 更新日志

本文件记录该项目的重要变更。

## 0.14.0 - 2026-07-23

本版本聚焦技能资源库工作流、来源更新可靠性、设置页整理和更统一的界面表现。

### 新功能

- 新增统一的技能资源库导入菜单，合并 GitHub 仓库导入和支持的 skills.sh 技能链接导入。
- GitHub-backed 技能导入到技能资源库时支持预览、选择、冲突处理和来源信息记录。
- 允许编辑、删除和恢复内置软件平台，并把修改持久化到本地配置。
- 设置页关于区域新增检查更新，可对比当前安装版本与 GitHub 最新发布版本。

### 改进

- 移除独立的技能市场页面和相关前端市场浏览界面。
- 技能资源库目录视图改为围绕 `作者/项目` 分组，而不是只显示作者列表。
- 技能卡片同时显示创建日期和更新日期，并按中英文界面使用对应语言标签。
- 只要技能具备可恢复的 GitHub 来源信息，即使缺少直接来源 URL，也显示单个技能的来源更新操作。
- 优化技能详情右侧栏，将备注、标签、来源信息、时间信息、存储路径、安装状态和技能集合整理为清晰分区。
- 设置页提示文案改为提示图标悬浮显示，并把目录保存按钮与对应路径输入框放在同一行，重新组织备份和 WebDAV 控件。
- 软件平台按龙虾类和编程类分组，每个分组内部保持响应式两列布局。
- 设置页区分已检测到 skills 目录和未检测到目录的内置平台；主侧边栏默认隐藏未检测到目录的内置平台。
- 主题和设置入口移动到左下角，主题中的“跟随系统”重命名为“系统”，移除顶部全局搜索框，并加宽展开后的侧边栏。
- 刷新中英文 README 截图，确保每份文档使用对应语言界面截图。
- 统一跨平台界面字体栈，降低字体割裂感。

### 修复

- 移除技能市场界面后保留 GitHub 导入能力。
- 修复技能资源库“从来源更新”错误更新中央技能库、导致资源库列表被清空的问题。
- 从已保存的 GitHub 仓库和路径信息恢复缺失的来源 URL，再执行来源更新。
- 来源更新覆盖本地 `SKILL.md` 前会校验下载内容，避免写入无效文件。
- 避免 GitHub 导入的技能因来源信息不完整而失去后续更新能力。
- 内置平台修改后可在应用重启后恢复。
- 减少技能详情右侧栏中的重复和未分组元数据。

## 0.13.0 - 2026-07-22

本版本聚焦内部发布准备和打包元数据。

## 0.12.0 - 2026-07-20

本版本聚焦软件平台管理、项目发现行为和分平台打包脚本。

### 新功能

- 将设置页中的扫描目录和自定义平台整合为更清晰的平台与项目目录管理区域。
- 将管理对象拆分为软件平台和项目目录，并支持查看内置软件平台。
- 新增 Windows、macOS、Linux 三个独立发布打包脚本，每个脚本只负责自己的宿主平台。

### 改进

- 本地不存在对应 skills 目录时，主界面不再显示该内置软件平台。
- 设置页默认折叠内置软件平台，让平台管理区域更紧凑。
- 优化设置页布局，把新增操作放到对应分区标题旁边。
- 更新中英文设置页截图以匹配当前界面。
- 优化 README 中的打包说明，改为平台专用命令和统一参数表。
- 停止追踪 `docs/` 下的本地规划文档，并从仓库移除生成型 `release-notes` 文件。

### 修复

- 未配置项目目录时，浏览器 fixture 中的项目技能不再出现在项目技能库。
- 项目技能库只显示来自已配置项目目录的结果；没有启用的项目目录时不再展示旧缓存项目技能。
- 修复 Homebrew Rust 覆盖 rustup toolchain 时 macOS universal 打包找不到目标标准库的问题。
- 修复 macOS 和 Linux 打包脚本中 `mktemp` 模板的跨平台兼容性。
